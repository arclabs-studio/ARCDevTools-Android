# ARCDevTools-Android

Centralized quality tooling, Claude Code automation, and development standards delivery for **ARC Labs Studio** Android projects.

> **This is a tooling repository**, not an application. It is integrated as a Git submodule in downstream Android projects.

## What This Repo Does

ARCDevTools-Android delivers three categories of tooling to downstream projects:

### 1. Code Quality

| Tool | Config | Purpose |
|------|--------|---------|
| **ktlint** | `configs/.editorconfig` | Kotlin code style enforcement (120 chars, Compose rules) |
| **detekt** | `configs/detekt.yml` | Static analysis with all rule categories |
| **detekt-compose** | `configs/detekt-compose.yml` | Jetpack Compose-specific static analysis |

### 2. Git Automation

| Hook | Location | Trigger |
|------|----------|---------|
| **pre-commit** | `hooks/pre-commit` | ktlint format + check, detekt on staged files |
| **pre-push** | `hooks/pre-push` | Run unit tests before push |

### 3. Claude Code Integration

| Component | Location | Purpose |
|-----------|----------|---------|
| **format-on-save** | `claude-hooks/format-on-save.sh` | Auto-format .kt/.kts with ktlint after Edit/Write |
| **validate-commit** | `claude-hooks/validate-commit.sh` | Enforce Conventional Commits on `git commit` |
| **block-dangerous-git** | `claude-hooks/block-dangerous-git.sh` | Block force push, reset --hard, clean -f, branch -D |
| **session-start** | `claude-hooks/session-start.sh` | Show branch, changes, last release on session start |
| **settings.json** | `.claude/settings.json` | Hook registrations for all 4 hooks |
| **setup-skills** | `scripts/setup-skills.sh` | Symlink ARCKnowledge-Android skills to .claude/skills/ |
| **setup-mcp** | `scripts/setup-mcp.sh` | Register MCP servers (Context7, Android Skills) |
| **MCP template** | `configs/mcp/claude-mcp.json` | Reference MCP config for downstream projects |

## How Downstream Projects Use This

```bash
# 1. Add as submodule
git submodule add https://github.com/arclabs-studio/ARCDevTools-Android
git submodule update --init --recursive

# 2. Run setup (copies configs, hooks, Claude integration)
./ARCDevTools-Android/arc-setup

# 3. Optional: configure MCP servers
./ARCDevTools-Android/arc-setup --with-mcp
```

`arc-setup` copies/generates into the downstream project:

```
PROJECT_ROOT/
├── .editorconfig              ← copied from configs/
├── config/detekt/
│   ├── detekt.yml             ← copied from configs/
│   └── detekt-compose.yml     ← copied from configs/
├── .git/hooks/
│   ├── pre-commit             ← copied from hooks/
│   └── pre-push               ← copied from hooks/
├── claude-hooks/              ← copied from claude-hooks/
│   ├── session-start.sh
│   ├── validate-commit.sh
│   ├── block-dangerous-git.sh
│   └── format-on-save.sh
├── .claude/
│   ├── settings.json          ← copied from .claude/
│   └── skills/                ← symlinked from ARCKnowledge-Android
├── Makefile                   ← auto-generated
└── ARCDevTools-Android/       ← this submodule
```

## Makefile Commands

Generated by `arc-setup` in downstream projects:

```bash
make lint      # Run detekt and ktlint
make format    # Check formatting (dry-run)
make fix       # Apply ktlint format
make test      # Run unit tests
make setup     # Re-run ARCDevTools setup
make hooks     # Re-install git hooks
make skills    # Re-link Claude Code skills
make mcp       # Configure MCP servers
make clean     # Clean build artifacts
```

## arc-setup Flags

```bash
./ARCDevTools-Android/arc-setup                    # Interactive mode
./ARCDevTools-Android/arc-setup --with-workflows   # Install GitHub Actions (non-interactive)
./ARCDevTools-Android/arc-setup --no-workflows     # Skip GitHub Actions
./ARCDevTools-Android/arc-setup --with-mcp         # Configure MCP servers (non-interactive)
./ARCDevTools-Android/arc-setup --no-mcp           # Skip MCP configuration
./ARCDevTools-Android/arc-setup --help             # Show help
```

## Claude Code Hooks Detail

### PostToolUse: format-on-save.sh

Triggers on every Edit/Write of `.kt`/`.kts` files. Runs `ktlint --format` on the file. Requires ktlint CLI installed (`brew install ktlint` or download from GitHub releases). Degrades gracefully if ktlint is not found.

### PreToolUse: validate-commit.sh

Intercepts `git commit` commands. Validates commit message matches Conventional Commits: `type(scope): subject`. Valid types: feat, fix, docs, style, refactor, perf, test, chore, build, ci, revert.

### PreToolUse: block-dangerous-git.sh

Blocks destructive git operations: force push to main/develop, `git clean -f`, `git reset --hard` (except `origin/*`), `git branch -D main/develop`. Strips heredocs and quoted strings before matching to avoid false positives.

### SessionStart: session-start.sh

Shows current branch, pending changes count, last release tag, uncommitted/staged stats. Requires `jq`.

### Stop: Compilation Check

Prompt hook (runs on claude-haiku-4-5) that verifies the project compiles via `./gradlew assembleDebug` if Kotlin files were edited during the session.

## MCP Servers

`scripts/setup-mcp.sh` auto-registers (idempotent):

| Server | Package | API Key Required |
|--------|---------|-----------------|
| **Context7** | `@upstash/context7-mcp@latest` | No |
| **Android Skills** | `android-skills-mcp` | No |

Manual setup needed for:
- **GitHub**: `@anthropic/github-mcp@latest` (needs `GITHUB_TOKEN`)
- **Linear**: `@anthropic/linear-mcp@latest` (needs `LINEAR_API_KEY`)

Full template at `configs/mcp/claude-mcp.json`.

## Relationship to ARCKnowledge-Android

| Repo | Role | Contains |
|------|------|----------|
| **ARCKnowledge-Android** (submodule) | Knowledge | Skills, agents, architecture docs, quality standards |
| **ARCDevTools-Android** (this repo) | Delivery | Configs, hooks, scripts, CI workflows, setup automation |

ARCKnowledge-Android is embedded as a Git submodule. `setup-skills.sh` symlinks its `.claude/skills/` into the downstream project so Claude Code can discover them.
